# 写真台帳アプリ（統合版）ver.2.0

これまで別々だった **「写真台帳 照合アプリ（デジタル看板用）」** と **「手書き写し込み台帳アプリ（手書き看板用）」** を、**1本の `index.html`** にまとめたものです。起動時に **モードを選ぶ** だけで、どちらの方式でも使えます。

- 🖥 **デジタル看板モード** … スマホの「写しこみ看板」入力アプリで作った **QR を読み取って** 台帳化（＝従来の照合アプリと同じ動作）
- ✍ **手書き看板モード** … 識別QRを貼った看板の **手書き内容を OCR**（クラウド高精度／端末内）で読み取り（＝従来の手書き写し込み台帳と同じ動作）

CSV／Excel出力・作業の保存と復元・写真振り分け・看板マスタ（選択肢）編集は、**両モード共通**で使えます。

---

## 中身（リポジトリ直下用）

この一式は「リポジトリの直下」または GitHub Pages の公開フォルダに **そのまま置く** 前提です（フォルダは作りません）。

```
index.html                          統合アプリ本体（名前を変えないでください）
opencv.js                           QR検出の強化用（大きいQRや傾き対策。無くてもjsQRで動作）
tesseract.min.js / worker.min.js    端末内OCR（オフライン時のフォールバック）
tesseract-core-simd-lstm.wasm(.js)  同上のコア
tessdata/jpn.traineddata.gz         日本語OCR辞書
tessdata/eng.traineddata.gz         英数字OCR辞書
看板識別QR_印刷シート.html          手書きモード用の「識別QR（内容 "T1"）」印刷シート
marker_qr.png                       識別QRの単体画像（予備）
ocr-proxy/worker.js                 手書きモードのクラウド高精度OCR 中継サーバー（Cloudflare Workers）
ocr-proxy/wrangler.toml             同上の設定
README.md                           このファイル
```

`index.html` は同じフォルダ内の `opencv.js` や `tesseract*` を相対パスで読みます。**ファイル名・配置は変えないでください。**

## 使い方（共通）

1. `index.html` を開くと、最初に **モード選択画面** が出ます。「🖥 デジタル看板」か「✍ 手書き看板」を選びます。
   - あとからでも、ヘッダー右上の **「🔀 モード変更」** でいつでも切り替えられます。
2. **STEP1**「フォルダを選択」で写真を取り込みます（ドラッグ＆ドロップも可）。QR付きの看板写真が **区切り** になり、看板ごとにグループ化されます。
3. **STEP2** で各グループの内容を確認・修正します（「看板を編集」）。
4. **STEP3**（またはヘッダー）から **台帳CSV出力／台帳Excel出力** で書き出します。
5. 必要に応じて **写真振り分け**（コピー＆リネーム）、**作業を保存／復元**（JSON）も使えます。

## モード別のポイント

### 🖥 デジタル看板モード

現場でスマホの「写しこみ看板」入力アプリに件名・対象などを入力し、画面に出た **QRを撮影** しておきます。このアプリはそのQRの中身をそのまま読み取って台帳にします。

- QRの形式は `T1｜件名｜調査区分｜対象1｜対象2｜状況｜方向｜撮影日｜撮影者｜種別｜備考` の11項目です。
- 手書きOCR関連のボタン（中継URL設定・看板を手書きOCR・高精度OCRトグル）は、このモードでは表示されません。
- QRが読めなかった写真は「OCRでグループ作成」（端末内Tesseract）で補完するか、手動でグループを作れます。

### ✍ 手書き看板モード

看板に貼った **識別QR（内容 "T1"）** を「これは看板」という目印として使い、**手書きの内容はOCRで読み取ります**。

- 初回だけ `看板識別QR_印刷シート.html` を開いて印刷し、看板に貼ってください（ラミネートやマグネットにすると繰り返し使えます）。
- **高精度OCR（手書き対応）を使うには、中継サーバー（プロキシ）のURLを一度だけ登録** します（STEP1の「中継URL設定」または操作バーの「中継URL」）。APIキーは中継サーバー側に置くので、この `index.html` には出ません。送信されるのは看板写真のみです。
- ネットに繋がらない現場では、自動で **端末内Tesseract** に切り替わります（手書き精度は落ちます）。
- 件名・撮影者・撮影日は、空欄なら前の看板から自動で前送り補完します。

## 中継サーバー（手書きモードのクラウド高精度OCR）

`ocr-proxy/` は Cloudflare Workers 用の中継サーバーです。**手書きモードで高精度OCRを使う場合だけ** 必要です（デジタル看板モードだけを使うなら不要）。

必要な環境変数（シークレット）:

- `GEMINI_API_KEY` … Google AI Studio で発行したキー
- `GEMINI_MODEL` … 省略可。既定 `gemini-2.5-flash`
- `ALLOW_ORIGIN` … 省略可。既定 `*`。自分の GitHub Pages のURLに絞ると安全

デプロイ後に表示されるURLを、アプリの「中継URL設定」に登録してください。

## 従来アプリとの関係

- **写しこみ看板 入力アプリ（スマホ用PWA・`/utusikomi/`）** … デジタル看板モードで読む **QRを“作る”** 現場用ツール。今回の統合には含めていません（スマホ側で別デプロイのまま使ってください）。
- 本統合版は、**PC側で“読む”** 2アプリ（照合＋手書き）を1本化したものです。デジタル看板モードは照合アプリ ver.1.03、手書き看板モードは手書き写し込み台帳 ver.1.14 と同じ挙動になるよう作っています。
