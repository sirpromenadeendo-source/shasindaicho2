# 写真台帳アプリ（統合版）ver.2.1 ― Pages配置用（スリム版）

GitHub Pages／リポジトリ直下に置く **最小構成** です。アプリ本体の動作に必要なファイルだけを入れています。
（`ocr-proxy/`・`看板識別QR_印刷シート.html`・`marker_qr.png` は含みません。→ 末尾「別途管理するもの」参照）

これまで別々だった **「写真台帳 照合アプリ（デジタル看板用）」** と **「手書き写し込み台帳アプリ（手書き看板用）」** を、**1本の `index.html`** にまとめたものです。起動時に **モードを選ぶ** だけで、どちらの方式でも使えます。

- 🖥 **デジタル看板モード** … スマホの「写しこみ看板」入力アプリで作った **QR を読み取って** 台帳化（＝従来の照合アプリと同じ動作）
- ✍ **手書き看板モード** … 識別QRを貼った看板の **手書き内容を OCR**（クラウド高精度／端末内）で読み取り（＝従来の手書き写し込み台帳と同じ動作）

CSV／Excel出力・作業の保存と復元・写真振り分け・看板マスタ（選択肢）編集は、**両モード共通**で使えます。

---

## 中身（この一式を「リポジトリ直下」または Pages 公開フォルダにそのまま置く）

```
index.html                          統合アプリ本体（名前を変えないでください）
opencv.js                           QR検出の強化用（大きいQRや傾き対策。無くてもjsQRで動作）
tesseract.min.js / worker.min.js    端末内OCR（オフライン時／QR不読時のフォールバック）
tesseract-core-simd-lstm.wasm(.js)  同上のコア
tessdata/jpn.traineddata.gz         日本語OCR辞書
tessdata/eng.traineddata.gz         英数字OCR辞書
README.md                           このファイル
```

`index.html` は同じフォルダ内の `opencv.js` や `tesseract*` を相対パスで読みます。**ファイル名・配置は変えないでください。**

---

## 使い方（共通）

1. `index.html` を開くと、最初に **モード選択画面** が出ます。「🖥 デジタル看板」か「✍ 手書き看板」を選びます。
   - あとからでも、ヘッダー右上の **「🔀 モード変更」** でいつでも切り替えられます。
2. **STEP1**「フォルダを選択」で写真を取り込みます（ドラッグ＆ドロップも可）。QR付きの看板写真が **区切り** になり、看板ごとにグループ化されます。
3. **STEP2** で各グループの内容を確認・修正します（「看板を編集」）。
4. **STEP3**（またはヘッダー）から **台帳CSV出力／台帳Excel出力** で書き出します。
5. 必要に応じて **写真振り分け**（コピー＆リネーム）、**作業を保存／復元**（JSON）も使えます。

## モード別のポイント

### 🖥 デジタル看板モード
現場でスマホの「写しこみ看板」入力アプリに件名・対象などを入力し、画面に出た **QRを撮影** しておきます。このアプリはそのQRの中身をそのまま読み取って台帳にします。QRの形式は `T1｜件名｜調査区分｜対象1｜対象2｜状況｜方向｜撮影日｜撮影者｜種別｜備考` の11項目です。QRが読めなかった写真は、選択して「OCRでグループ作成」で補完でき、**高精度OCR（クラウド）を設定していれば手書きモードと同じクラウドOCRで読み取ります**（ver.2.1〜）。一括の「看板を手書きOCR」ボタンは手書きモード専用のため表示されません。

### ✍ 手書き看板モード
看板に貼った **識別QR（内容 "T1"）** を「これは看板」という目印として使い、**手書きの内容はOCRで読み取ります**。高精度OCR（手書き対応）を使うには、中継サーバー（プロキシ）のURLを一度だけ登録します（APIキーは中継サーバー側に置くので `index.html` には出ません）。ネット非接続時は自動で端末内Tesseractに切り替わります。

## 別途管理するもの（このスリム版には含みません）

- **`看板識別QR_印刷シート.html` ／ `marker_qr.png`** … 手書きモードで看板に貼る識別QR（内容 "T1"）を印刷するためのシートと予備画像。手元に保管し、必要な人が印刷して使ってください（アプリ本体からは参照しないので、Pagesに置く必要はありません）。
- **`ocr-proxy/`（worker.js / wrangler.toml）** … 手書きモードのクラウド高精度OCR 中継サーバー（Cloudflare Workers）。GitHub Pagesでは動かないため、**Cloudflareに別デプロイ**します。ソースは手元か別リポジトリで管理してください。
  - `GEMINI_API_KEY` は必ず `npx wrangler secret put GEMINI_API_KEY` で登録（ファイルに書かない）。
  - `ALLOW_ORIGIN` は自分の `https://ユーザー名.github.io` に絞ると安全（既定 `*` は誰でも呼べてしまう）。

> フル版（印刷シート・中継サーバーのソースも同梱）が必要なときは、別途配布の `写真台帳アプリ_統合版.zip` を使ってください。
